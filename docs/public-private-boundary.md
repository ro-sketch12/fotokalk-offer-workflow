# Public/Private Boundary · FotoKalk

## Öffentlich

- Projektbeschreibung
- Architektur auf Komponentenebene
- Workflow vom Setup bis zur Prüfung
- Tech-Stack als Liste
- bekannte Grenzen
- Links zu geprüften Portfolio-/Demo-Oberflächen

## Privat

- `.env`
- API-Keys, Tokens, Cookies, Credentials
- Auth-, Admin-, Team-, Billing- und Payment-Details
- echte Kundendaten, Angebote, Rechnungen, Kontakte
- produktive Datenbankmigrationen
- interne Prompts und genaue API-Routen
- private Screenshots

## Formulierung

Gut:

- “KI-gestützte Vorbereitung”
- “prüfbarer Angebotsentwurf”
- “fachliche Kontrolle bleibt beim Nutzer”

Vermeiden:

- absolute Automatisierungs-, Compliance- oder Genauigkeitsaussagen ohne aktuelle Prüfung
- harte Leistungs- oder Geschäftsergebnisversprechen ohne Messung
- Formulierungen, die so wirken, als würde die App fachliche Verantwortung übernehmen
